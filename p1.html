Sales Analysis

Embedded PowerBi Environment



Details:

Problem Statement: The sales director wanted to have the sales insights of the organization so that he can track the revenue and take actions on the locations which aren't doing well. He wanted to interpret the data based on the locations particularly to work on those regions with that department head.

This project includes extracting data from databases by connecting mysql to powerbi. Data is in different tables so the tables are connected in star schema with the fact table as main priority and detailing it with multiple tables around. This project is an end on end project connected from one page to another page with buttons. Buttons on the top of the page are labeled in icons and are connected to a particular page.

Dashboard includes 3 pages: Home page , Reports, Details. In the first page it's a simple home screen connected to the next 2 pages. The second page contains insights of the data and the third page contains the tabular details of overall data used in the dashboards. Transformations: Changing the formats , Adding new measures, standardizing the currency values into rupees so that there won't be any value differences, deleting the duplicate rows, and columns.Row level security is used for this project so that the zonal head of a particular zone can only view the details necessary for him instead of whole insight.
